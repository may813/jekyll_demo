  1. 清除浮动的方法
一个父元素高度自适应，内部有浮动元素时，父元素高度无法撑开
方法一：
为父元素设置overflow为visible以外的值，如：auto,hidden,scroll
为父元素设置display

  2. BFC块格式化上下文
是一个区域，规定了内部块盒的渲染方式，以及浮动相互之间的影响关系。
a) BFC会隔离内部与外部，其渲染相互之间不影响
b) BFC区域不会和外部浮动盒子外边距发送叠加
c) BFC计算高度的时候内部浮动元素的高度也会计算在内

    创建BFC的方式： 浮动元素（float不是none）、绝对定位元素(absolute或fixed)、行内块元素（display:inline-block)、
overflow(值不为visible)、display(flow-root)、contain(layout、content、strict)、
弹性元素（display为flex或inline-flex元素的直接子元素）、网格元素（display为grid或inline-grid元素的直接子元素）,zoom:1(IE)

解决：
浮动高度无法撑开，外边距合并(垂直方向边距会叠加)

  3. 布局自适应的方式
a) 单位使用rem和em，都是基于字体大小的相对单位，em使用当前元素的字体大小确定实际尺寸，而rem是根据html根节点的字体大小
b) 浮动，inline-block
c) flex
d) @media的查询条件，针对不同尺寸的容器使用不同的样式
e) 不使用固定宽高，使用百分比

  4. 伪类与伪元素的区别
伪类定义某元素的某种状态或者筛选
:hover,:active,:focus,:visited,:link,
:root,:not,:first-child,:last-child,:only-child,:only-of-type,:empty,:enabled,:disabled,:checked
:required,:valid,:invalid
:nth-child(n),:nth-of-type(n)

伪元素不存在于DOM树中的虚拟元素，可以像正常的html元素一样定义css,但是无法使用js获取
::before,::after,
::first-letter,::first-line
::selection

  5. 盒模型
主要包括margin,padding,content,border，使用box-sizing可以修改width属性作用的范围
box-sizing: padding-box
box-sizing: content-box W3C标准
box-sizing: border-box IE
块级元素margin,padding均有效，但是外边距会叠加
内联元素margin-left,margin-right,padding-left,padding-right有效，上下的margin,padding无效
块级元素：div,h1\h6,hr,ol,ul,li,p

  6. 常见兼容性
a) 不同浏览器的标签默认margin,padding不同
b) IE6双边距bug，块级元素浮动，且设置了水平margin，其margin翻倍，解决：display:inline
c) 不同浏览器设置不同的样式
{
	background-color:#f1ee18;/*所有识别*/
	.background-color:#00deff\9; /*IE6、7、8识别*/
	+background-color:#a200ff;/*IE6、7识别*/
	_background-color:#1e0bd1;/*IE6识别*/
}

  7. 定位postion
fixed,absolute都会使元素脱离文档流
fixed: 元素相对于浏览器窗口的位置，若祖先元素设置了transform则相对于祖先元素移动
releate: 元素相对于自身位置偏移, 元素不脱离文档流，不影响后面元素的布局
absolute: 相对于距该元素最近的已定位的祖先元素
static: 默认值
inherit: 继承，IE8不支持

  8. 继承属性
会继承的属性: font-size, font-family, color
不会继承的属性: border, padding, margin, width, height

  9. 水平、垂直居中
水平居中：text-align:center, margin: 0 auto, absolute+transform:translateX(), absolute+margin-left(已知宽度), flex+justify-content, 
垂直居中：line-height(已知高度), absolute+transform:translateY(), absolute+margin-top(已知高度), flex+align-items, flex+align-self(子元素), table
水平垂直居中: absolute+transform::translate(), absolute+margin-top+margin-left, flex+justify-content+align-items

  11. 浏览器怎么解析css选择器
CSS选择器的解析是从右向左，先找到所有最右节点，遍历每个节点，向上找父节点，直到根元素或者满足条件的匹配规则，则结束这个节点的遍历。
